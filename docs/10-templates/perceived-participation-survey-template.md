# Perceived Participation Survey Template

Use a 1–5 scale.

| Statement | Score |
|---|---|
| I understood why this security change was introduced. |  |
| I had an opportunity to provide feedback. |  |
| My team’s operational needs were considered. |  |
| The final process is practical for my work. |  |
| I know how to request help or an exception. |  |
| Communication before rollout was sufficient. |  |
| The security change improves protection without unnecessary friction. |  |

## Free-text

- What should be improved?
- What risk or workaround do you see?
- What additional guidance would help?

## Usage guidance

Use this template to document **Perceived Participation Survey**.

## Evidence to retain

Retain the approved record, source evidence, approval history, exceptions, and follow-up actions under the organization's retention and protection rules.

## Practical example

For one in-scope service, the owner completes the **Perceived Participation Survey** record, links authoritative evidence, obtains review, and tracks rejected or incomplete items to closure.

## ISO requirement, implementation guidance, and best practice

This exact template is not an ISO requirement; it is guidance for recording **Perceived Participation Survey** consistently. Controlled values, named owners, timestamps, approvals, and authoritative evidence improve assurance.
