# Change Enablement Security Controls

Change enablement is one of the strongest links between ITSM and ISO 27001.

## Security fields to add to change records

- affected service or asset
- data classification
- security impact
- privacy impact
- access-control impact
- logging/monitoring impact
- vulnerability impact
- supplier impact
- rollback plan
- security testing performed
- risk acceptance required?
- post-implementation review required?

## Example

A firewall rule change allowing partner API access should include business justification, partner identity, data classification, allowed source/destination, expiration or review date, logging requirement, security approval, and rollback plan.

## Best practices

- Use standard changes for low-risk repeatable work.
- Require enhanced review for high-risk changes.
- Include emergency change review after the fact.
- Link change records to risk and incident records where appropriate.
- Sample changes during internal audit.
