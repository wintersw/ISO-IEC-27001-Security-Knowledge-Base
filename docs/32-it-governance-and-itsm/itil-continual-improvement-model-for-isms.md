# ITIL Continual Improvement Model for ISMS

A practical improvement model asks:

1. What is the vision?
2. Where are we now?
3. Where do we want to be?
4. How do we get there?
5. Take action.
6. Did we get there?
7. How do we keep momentum?

## Example: improve vulnerability remediation

### Vision

Critical vulnerabilities on internet-facing systems are remediated before exploitation creates customer harm.

### Current state

Only 62% are remediated within target; asset ownership is incomplete.

### Target state

95% within target, 100% ownership, all exceptions approved and time-limited.

### Plan

Improve CMDB ownership, integrate scanning with tickets, define escalation, and add verification.

### Verify

Measure target achievement, exception quality, scanner coverage, and repeated findings.

## Best practices

- Define a measurable target.
- Understand current-state evidence.
- Avoid launching tools before understanding process causes.
- Verify outcomes after implementation.
- Maintain momentum through governance and backlog review.
