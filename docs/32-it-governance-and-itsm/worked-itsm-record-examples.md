# Worked ITSM Record Examples

## High-risk change example

**Change:** Enable partner API access to customer order status.

The record should include affected service, partner identity, data fields, authentication method, authorization rules, rate limits, logging, test results, rollback plan, approval, and post-implementation review.

## Security incident example

**Incident:** Suspicious bulk export from support account.

The record should include detection source, severity, timeline, affected data, containment, evidence IDs, escalation, notification decision, recovery, and lessons learned.

## Problem record example

**Problem:** Repeated terminated-user access in SaaS applications.

The record should include incidents/findings, root cause, workaround, permanent correction, risk, owner, due date, and effectiveness criteria.

## Service request example

**Request:** Temporary production database access.

The record should include business reason, exact privilege, duration, approvals, MFA, monitoring, expiry, and removal evidence.

## Configuration item example

**CI:** Production customer database.

The record should include owner, service, environment, classification, criticality, recovery objective, backup, logging, patch group, supplier, and dependencies.
