# Cybersecurity Supply-Chain Governance

Supply-chain risk is broader than vendor questionnaires. It includes software dependencies, cloud services, managed service providers, subcontractors, data processors, open-source components, and critical technology suppliers.

## Governance questions

- Which suppliers support critical services?
- Which suppliers process restricted data?
- What security outcomes are required in contracts?
- How are material changes and incidents reported?
- How are fourth parties managed?
- What evidence is reviewed during operation?
- What happens at termination?
- Can the organization continue if the supplier fails?

## Example

A managed service provider has privileged access to production. Governance should require named accountability, strong authentication, session monitoring, access review, breach notification, continuity commitments, subcontractor controls, and exit support.

## Best practices

- Tier suppliers by criticality and data exposure.
- Include security before procurement approval.
- Monitor supplier change, not only annual certification.
- Define risk acceptance for unresolved supplier findings.
- Maintain exit and substitution plans for critical dependencies.
