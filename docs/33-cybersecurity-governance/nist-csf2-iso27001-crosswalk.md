# NIST CSF 2.0 and ISO 27001 Crosswalk

The two frameworks serve different purposes but work well together. ISO/IEC 27001 defines requirements for an information security management system (ISMS). NIST CSF 2.0 organizes cybersecurity outcomes in a language that is useful for strategy, profiles, communication, and capability planning.

| NIST CSF 2.0 Function | ISO/IEC 27001 relationship |
|---|---|
| Govern | Clauses 4–7, management review, risk strategy, supplier governance |
| Identify | context, scope, assets, risk assessment |
| Protect | Annex A preventive and protective controls |
| Detect | logging, monitoring, threat intelligence, event assessment |
| Respond | incident planning, assessment, response, communication |
| Recover | continuity, recovery, lessons learned, improvement |

## Practical use

An organization can use ISO 27001 as the certifiable management-system backbone and NIST CSF 2.0 Profiles to communicate current and target cybersecurity outcomes.

## Example

The organization creates a target profile for critical services. Gaps are translated into risk-treatment actions, Annex A control improvements, IT Service Management (ITSM) changes, and management-review decisions.

## Best practices

- Do not claim one framework automatically proves conformity with the other.
- Map outcomes to existing evidence rather than creating duplicate evidence.
- Use CSF Profiles for roadmap communication.
- Use ISO 27001 internal audit to test management-system operation.


## Evidence to retain

Retain records showing both design decisions and actual operation, such as:

- approved strategy or governance decision
- authority and accountability record
- risk and performance reporting
- oversight minutes and tracked actions
