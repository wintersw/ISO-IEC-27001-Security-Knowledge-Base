# Security Governance Model

A governance model defines how security decisions are made, approved, monitored, and improved.

## Governance layers

| Layer | Purpose |
|---|---|
| Board / executive oversight | Direction, risk appetite, major decisions |
| Security steering committee | Priorities, resources, risk decisions |
| Risk and control owners | Day-to-day accountability |
| Security operations | Detection, response, technical control operation |
| Internal audit | Independent assurance |

## Decision types

- risk acceptance
- exception approval
- control prioritization
- security objectives
- budget and resource allocation
- policy approval
- corrective action escalation

## Best practices

- Define decision rights.
- Retain minutes and action logs.
- Use risk language rather than tool language.
- Escalate overdue actions.
- Review metrics that inform decisions.
