# Three Lines Model for Security Assurance

The three lines model helps separate operational ownership, oversight, and independent assurance.

## First line — Operational management

Owns and operates risks and controls.

Examples: system owners, engineering teams, service owners, process owners.

## Second line — Oversight and specialist functions

Provides policy, guidance, monitoring, and challenge.

Examples: information security, GRC, privacy, risk management, compliance.

## Third line — Independent assurance

Provides independent evaluation.

Examples: internal audit, external audit, certification audit.

## Practical security-team implication

Security may be first line for security tools, second line for enterprise security governance, or both depending on organization size. Documenting role boundaries prevents conflicts and improves auditability.
