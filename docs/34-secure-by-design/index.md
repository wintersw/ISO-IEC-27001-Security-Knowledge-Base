# Secure by Design

Secure by Design treats customer security outcomes as a core product requirement. It asks product manufacturers and service providers to reduce avoidable security burden instead of requiring every customer to compensate for unsafe defaults.

This concept complements ISO/IEC 27001 because many Annex A controls depend on how products are designed, configured, updated, monitored, and supported.

## Three governance themes

1. **Take ownership of customer security outcomes.**
2. **Use transparency and accountability to improve trust.**
3. **Make secure features available by default rather than as expensive add-ons.**

## Example

A software as a service (SaaS) product should not require customers to buy a premium tier to obtain multifactor authentication (MFA), security logs, or secure default settings for basic risk reduction. Product design should make common secure choices easy and unsafe choices visible.
