# Evidence by Design

## Guidance

- Design workflows that automatically retain requester, approver, timestamp, decision, and outcome.
- Use tickets, logs, reports, and dashboards as normal operating evidence.
- Avoid manual audit evidence collection where possible.

## Why this matters

Security teams fail when processes depend on heroics, memory, or manual spreadsheet work. Sustainable information security management system (ISMS) operation requires integrated workflows, clear ownership, reliable evidence, and management attention.

## Checklist

- [ ] Owner assigned
- [ ] Workflow defined
- [ ] Evidence produced automatically where possible
- [ ] Metrics defined
- [ ] Exceptions tracked
- [ ] Reviewed periodically

## Detailed explanation

Evidence by design means deciding how control operation will be demonstrated before the control is launched. This reduces audit effort and improves daily management.

### Example

A supplier review workflow automatically records risk tier, reviewer, evidence, findings, approvals, due dates, and renewal date. The same record supports supplier governance, risk treatment, internal audit, and customer assurance.

### Best practices

- define evidence owner and source
- prefer automated system records
- retain complete populations
- link remediation and decisions
- use consistent identifiers
- protect sensitive evidence
- define retention
- test retrieval before audits
- map one evidence source to multiple requirements

### Common mistakes

Evidence is stored in personal folders, screenshots replace authoritative reports, records cannot be linked to assets or controls, and old evidence is deleted before the audit period ends.

## Practical example

A small security team pilots this practice in one high-risk workflow, measures whether it improves reliability or assurance, and expands it only after the result and operating cost are understood.

## Evidence to retain

Retain records showing both design decisions and actual operation, such as:

- pilot decision and success criteria
- workflow or configuration records
- before-and-after results
- decision to adopt, adapt, or stop the practice
