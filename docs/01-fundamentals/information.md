# Information as an Asset

Information is anything that has meaning and value to the organization. It can be structured data, unstructured documents, source code, verbal knowledge, configuration, credentials, records, drawings, procedures, or business know-how.

## Why this matters

ISO/IEC 27001 protects information in all forms. A security team that only protects servers and databases will miss important [information assets](../07-security-domains/asset-management.md) such as spreadsheets, software as a service (SaaS) exports, printed contracts, backup media, application programming interface (API) keys, and undocumented operational knowledge.

## Practical examples

- Customer records in a customer relationship management (CRM)
- Source code in a Git repository
- Employee salary files
- Network diagrams
- Encryption keys
- Incident response playbooks
- A senior engineer’s undocumented recovery knowledge

## Implementation guidance

1. Define information categories.
2. Identify where each category is stored, processed, and transmitted.
3. Assign owners.
4. Classify sensitivity and criticality.
5. Link information assets to business processes.
6. Define protection, retention, transfer, and disposal rules.

## Evidence

- asset register
- data inventory
- classification scheme
- retention schedule
- access-control records
- information transfer records
