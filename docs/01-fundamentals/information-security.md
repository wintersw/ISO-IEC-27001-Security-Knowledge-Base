# What Is Information Security?

Information security is the coordinated protection of information against unwanted disclosure, improper alteration, loss, destruction, and unacceptable interruption.

## Information is broader than data

Information can be digital, physical, verbal, or embedded in knowledge. Examples include customer records, source code, architecture knowledge, printed contracts, credentials, operational procedures, and employee expertise. See [Information as an Asset](information.md).

## Security is a business discipline

Security decisions should be based on the harm that a failure could cause, which is the purpose of [risk management](../05-risk-management/index.md). The same technical weakness may be tolerable in a disposable test system and unacceptable in a safety-critical production service.

## Security versus compliance

Compliance confirms that specified obligations are addressed. Security reduces relevant risk. The two overlap, but neither guarantees the other (see [Security vs Compliance](security-vs-compliance.md)). A compliant organization may still have weak controls, and a technically strong organization may fail an audit because governance or evidence is inadequate.

## Practical example

A team deploys multi-factor authentication (see [Identity and Access Management](../07-security-domains/iam.md)) but has no process for emergency accounts, access reviews, or prompt offboarding. The technology is useful, but the access-control system is incomplete.


## Evidence to retain

Retain records showing both design decisions and actual operation, such as:

- documented design or risk decision
- implemented safeguard or configuration record
- test, review, or monitoring result showing the intended outcome
