# Certification Project Plan

## Workstreams

- scope and context
- governance and roles
- risk management
- policies and procedures
- Annex A control implementation
- evidence collection
- internal audit
- management review
- corrective actions
- certification audit support

## Milestones

1. project kickoff
2. scope approved
3. methodology approved
4. first risk assessment completed
5. SoA approved
6. priority controls implemented
7. internal audit completed
8. management review completed
9. corrective actions closed
10. certification audit

## Best practices

- Build evidence throughout the project.
- Do not schedule certification before the ISMS has operated.
- Run a readiness review before Stage 1.
- Treat Stage 1 feedback seriously.
