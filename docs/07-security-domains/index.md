# Security Domains

Security domains group related capabilities. ISO controls often reference these domains, but teams should learn the domain concepts independently.

Use these guides to translate risk-treatment decisions into coherent capabilities rather than isolated controls. Each domain explains its purpose, practical implementation, useful evidence, metrics, common failure modes, and relationships to other domains.

Start with the domains that protect the organization's most important information and services. Assign accountable owners, define interfaces between domains, and assess outcomes across the complete lifecycle from design and operation through monitoring and improvement.

## How to use this section

Start with the overview of **Security Domains**, then follow the linked articles according to the decision or task at hand. Use the related templates to record decisions and the checklists to verify completion. Each linked article distinguishes formal ISO requirements from implementation guidance and optional best practice.
