# ITIL, ISO/IEC 20000, and ISMS

ITIL provides guidance for IT and digital service management. ISO/IEC 20000 provides requirements and guidance for a service management system. ISO/IEC 27001 provides requirements for an information security management system.

## Relationship

| Topic | ITIL | ISO/IEC 20000 | ISO/IEC 27001 |
|---|---|---|---|
| Main purpose | Service management best practice | Service management system requirements | Information security management system requirements |
| Certification focus | People/practice certifications and organizational use | Service management system certification | ISMS certification |
| Security role | Information security management practice supports services | Security requirements can be integrated into service management | Security is the primary management-system focus |
| Best use | Service workflows and value streams | Formal service-management system | Security governance, risk, and controls |

## Integration points

- incident management
- change enablement
- service continuity
- supplier management
- service-level management
- configuration management
- problem management
- information security management
- continual improvement

## Practical example

A change-management process can satisfy service-management needs while also supporting ISO/IEC 27001 controls for change management, access control, logging, secure development, and operational control.

## Best practices

- Do not duplicate workflows for ITIL and ISO 27001.
- Add security evidence requirements to existing service workflows.
- Link service risk to information security risk.
- Include security objectives in service reporting.
- Use continual improvement across both systems.

## Evidence

- change tickets
- incident records
- service review minutes
- configuration records
- security approval records
- service continuity tests
