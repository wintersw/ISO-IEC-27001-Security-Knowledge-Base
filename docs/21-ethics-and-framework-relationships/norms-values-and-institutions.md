# Norms, Values, Institutions, and Controls

Values describe what an organization considers important. Norms translate values into expectations. Institutions make expectations stable. Controls operationalize them.

## Example chain

| Layer | Example |
|---|---|
| Value | Protect customer trust |
| Norm | Customer information must be handled responsibly |
| Institution | Information security policy and governance committee |
| Control | Access reviews, encryption, logging, incident response |

## Why institutions matter

Institutions reduce uncertainty. They define roles, decision paths, sanctions, accepted behavior, and escalation paths. A mature ISMS is an institutionalized security process, not a collection of isolated documents.

## Security implication

A control works better when people believe the underlying value is legitimate. A password policy that feels arbitrary is resisted. A policy linked to customer protection, fraud prevention, and business continuity is easier to internalize.

## ISMS professional practice

- Connect policies to values.
- Explain the “why” behind controls.
- Ensure sanctions are fair and proportionate.
- Review norms when technology, law, or culture changes.
- Build routines so security becomes normal work.
