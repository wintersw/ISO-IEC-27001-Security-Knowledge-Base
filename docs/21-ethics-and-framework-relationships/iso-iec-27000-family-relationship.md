# ISO/IEC 27000 Family Relationship

The ISO/IEC 27000 family supports information security management, cybersecurity, privacy protection, risk management, and control implementation.

## Core relationship

| Standard | Main role |
|---|---|
| ISO/IEC 27000 | Current ISMS overview; earlier editions also provided the family vocabulary baseline |
| ISO/IEC 27001 | Certifiable information security management system (ISMS) requirements |
| ISO/IEC 27002 | Control implementation guidance |
| ISO/IEC 27005 | Information security risk management guidance |
| ISO/IEC 27017 | Cloud security guidance |
| ISO/IEC 27018:2025 | personally identifiable information (PII) protection in public cloud processor contexts |
| ISO/IEC 27701:2025 | Standalone privacy information management system requirements and guidance |
| ISO/IEC 27035 | Incident management guidance |
| ISO/IEC 27031:2025 | information and communication technology (ICT) readiness for business continuity |
| ISO/IEC 27036 | Supplier relationship security guidance |

## How to explain it simply

- **ISO/IEC 27000** explains the ISMS family overview. Edition-specific terminology must be checked against the current applicable source; ISO/IEC 27000:2018 is withdrawn and retained only for legacy traceability.
- **ISO/IEC 27001** says what the ISMS must do.
- **ISO/IEC 27002** helps design controls.
- **ISO/IEC 27005** helps design risk management.
- Other standards extend guidance for specific domains.

## Common mistake

Do not say an organization is “certified to ISO 27000.” Certification usually refers to ISO/IEC 27001. The family supports the ISMS but is not used in the same way.

## Evidence

- standards applicability register
- certification scope
- SoA
- risk methodology
- control design references


## Practical example

Leadership compares this perspective with the organization's legal duties, values, risk appetite, and existing management systems before deciding which practices to adopt and how to communicate them.
