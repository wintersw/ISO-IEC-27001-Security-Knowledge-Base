# ISO/IEC 27000 Family Relationship

The ISO/IEC 27000 family supports information security management, cybersecurity, privacy protection, risk management, and control implementation.

## Core relationship

| Standard | Main role |
|---|---|
| ISO/IEC 27000 | Overview, concepts, and vocabulary |
| ISO/IEC 27001 | Certifiable ISMS requirements |
| ISO/IEC 27002 | Control implementation guidance |
| ISO/IEC 27005 | Information security risk management guidance |
| ISO/IEC 27017 | Cloud security guidance |
| ISO/IEC 27018 | PII protection in public cloud processor contexts |
| ISO/IEC 27701 | Privacy information management extension |
| ISO/IEC 27035 | Incident management guidance |
| ISO/IEC 27031 | ICT readiness for business continuity |
| ISO/IEC 27036 | Supplier relationship security guidance |

## How to explain it simply

- **ISO/IEC 27000** explains language and concepts.
- **ISO/IEC 27001** says what the ISMS must do.
- **ISO/IEC 27002** helps design controls.
- **ISO/IEC 27005** helps design risk management.
- Other standards extend guidance for specific domains.

## Common mistake

Do not say an organization is “certified to ISO 27000.” Certification usually refers to ISO/IEC 27001. The family supports the ISMS but is not used in the same way.
