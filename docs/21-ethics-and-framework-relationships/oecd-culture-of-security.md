# OECD Culture of Security

OECD security guidance emphasizes a broad culture of security: awareness, responsibility, response, ethics, democracy, risk management, security design, security management, and reassessment.

## Culture-of-security principles

| Principle | ISMS interpretation |
|---|---|
| Awareness | People understand risks and their responsibilities |
| Responsibility | Everyone has a role in protecting information |
| Response | Vulnerabilities and incidents are handled cooperatively |
| Ethics | Interests of others are respected |
| Democracy / rights | Security measures respect rights and legitimate interests |
| Risk assessment | Risks are evaluated and kept within acceptable limits |
| Security design | Security is built into systems and processes |
| Security management | Security is managed systematically |
| Reassessment | Security adapts as conditions change |

## Practical use

Use these principles to test whether an ISMS is becoming a living culture or remaining a compliance artifact.

## Questions for management review

- Are people aware of their security responsibilities?
- Are controls proportionate and ethical?
- Do security measures respect privacy and rights?
- Are incidents and vulnerabilities handled cooperatively?
- Are security requirements built into design?
- Are risks reassessed after change?
