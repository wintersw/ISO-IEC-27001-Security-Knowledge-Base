# Standards Selection Decision Guide

Different frameworks are useful for different goals. Do not select a framework only because it is popular.

## Decision guide

| Goal | Most relevant framework |
|---|---|
| Certifiable information security management | ISO/IEC 27001 |
| Control implementation guidance | ISO/IEC 27002 |
| Information security risk method | ISO/IEC 27005 |
| IT service management improvement | ITIL |
| Certifiable service management system | ISO/IEC 20000 |
| Enterprise governance of information and technology | COBIT |
| Payment-card data security | PCI DSS |
| Security culture and broad responsibility | OECD-style security principles |

## Selection questions

- What problem are we solving?
- Is certification required or optional?
- Who is the audience?
- What is the scope?
- What evidence is needed?
- Are there customer or regulatory drivers?
- What resources are available?
- Which existing processes can be reused?

## Best practices

- Use ISO/IEC 27001 as the information security management system (ISMS) backbone.
- Use ISO/IEC 27002 for control design.
- Use ITIL/ISO 20000 for service workflows.
- Use COBIT for governance alignment.
- Use sector-specific standards only where relevant.
- Map once, reuse evidence many times.


## Practical example

Leadership compares this perspective with the organization's legal duties, values, risk appetite, and existing management systems before deciding which practices to adopt and how to communicate them.

## Evidence to retain

Retain records showing both design decisions and actual operation, such as:

- framework-selection rationale
- stakeholder and obligation analysis
- approved governance decision
- review showing the approach remains suitable

Intent documents are insufficient on their own; retain scoped operating records, approvals, exceptions, and verified follow-up.
