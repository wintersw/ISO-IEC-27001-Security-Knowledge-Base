# Data Security Maturity Model

## Maturity levels

| Level | Description |
|---|---|
| 1 Ad hoc | data locations and access are largely unknown |
| 2 Documented | basic inventory and classification exist |
| 3 Operating | lifecycle controls operate for critical data |
| 4 Measured | metrics, monitoring, and control testing exist |
| 5 Optimized | data security is automated, risk-based, and continuously improved |

## Capability areas

- data inventory
- classification
- ownership
- access governance
- encryption and key management
- masking and minimization
- DLP and monitoring
- retention and deletion
- breach readiness
- supplier data governance
- data security metrics
- AI/data science governance

## Assessment outputs

- maturity score
- risk themes
- priority improvement actions
- resource needs
- management review input


## Typical evidence

- approved policy, standard, procedure, or architecture record
- risk assessment or design review
- owner and role assignment
- implementation plan
- operating records
- monitoring records
- exception or waiver decisions
- test results
- audit records
- management review decisions
