# Data Security Governance Operating Model

Data security governance turns data protection from isolated controls into repeatable management practice.

## Governance forums

| Forum | Purpose |
|---|---|
| ISMS steering group | risk, resources, priorities, management decisions |
| data governance council | classification, ownership, quality, lifecycle |
| privacy/security review board | high-risk processing and design reviews |
| architecture review | platform, cloud, AI, integration, and data-flow design |
| incident review | breach lessons learned and control updates |

## Decision rights

Define who approves:

- new sensitive data collection
- data sharing
- data exports
- access to restricted data
- retention exceptions
- production data in test environments
- high-risk analytics
- AI training data use
- cross-border or third-party processing

## Governance outputs

- data risk register
- data classification register
- data flow maps
- exceptions register
- data security metrics
- management review inputs
- improvement backlog


## Typical evidence

- approved policy, standard, procedure, or architecture record
- risk assessment or design review
- owner and role assignment
- implementation plan
- operating records
- monitoring records
- exception or waiver decisions
- test results
- audit records
- management review decisions
