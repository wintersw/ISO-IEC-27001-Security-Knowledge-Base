# External Source Theme Gap Analysis

This historical review compared external source themes with repository coverage and identified areas for more explicit treatment. It does not make the source material normative, and the “added enhancement” column records the outcome of that review cycle rather than a current completeness guarantee.

## Gap analysis summary

| Source theme | Existing project coverage | Added enhancement |
|---|---|---|
| information security management system (ISMS) as ongoing process, not a project/product | Covered in continual improvement | Added operational framing and roadmap emphasis |
| Whole-organization implementation impact | Covered in governance and awareness | Added implementation workstreams and change-management integration |
| 14 ISMS building blocks | Partially covered across sections | Added consolidated building-block model |
| Governance/risk/compliance viewpoints | Partially covered | Added governance, risk, and compliance (GRC) operating model |
| Policy/process/procedure layering | Partially covered in documentation templates | Added process architecture and process catalog |
| Process descriptions with inputs, outputs, roles, interfaces | Not explicit enough | Added process description model and template |
| Performance/risk/compliance monitoring and key control indicator (KCI) | key performance indicator (KPI)/key risk indicator (KRI) covered | Added KCI and GRC monitoring model |
| Certification scope assurance | Scope covered | Added scope assurance and certification evidence checks |
| Regulatory readiness / NIS2-style obligations | Framework mapping covered | Added generic regulatory readiness operating model |
| Integrated management systems | Mentioned | Added practical integration guidance |
| Document-control maturity | Covered in templates | Added policy-document architecture and review records |
| Resource economics | Covered in BSI enhancement | Added implementation-cost and central-solution decision logic |

## Implementation rule

Where the project already had a mature chapter, this update adds cross-links rather than rewriting the same topic.


## Practical example

When a new external ISMS research source is proposed for integration, the maintainers repeat this exercise: each source theme is checked against the table above and existing chapters. A theme already covered by a mature chapter — such as continual improvement — receives only cross-links, while a genuinely new theme like KCI monitoring resulted in a new page. The "Added enhancement" column records that outcome, so later reviewers can see why each theme did or did not produce new content.

## Evidence to retain

When applying this review approach to your own ISMS, retain:

- the source-theme comparison table with coverage conclusions
- decisions to add, cross-link, or reject each theme with rationale
- links from accepted themes to the resulting pages or documents
