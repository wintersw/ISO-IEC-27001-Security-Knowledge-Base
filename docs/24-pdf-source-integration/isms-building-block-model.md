# ISMS Building-Block Model

A practical ISMS can be explained as a set of interconnected building blocks.

## Fourteen building blocks

| No. | Building block | Purpose |
|---:|---|---|
| 1 | Context of the organization | Understand internal/external context and interested-party requirements |
| 2 | Leadership and commitment | Ensure visible management direction and resources |
| 3 | Information security objectives | Translate business needs into measurable security objectives |
| 4 | Information security policy | Set the strategic policy anchor |
| 5 | Roles, responsibilities, and competencies | Define authority, accountability, skills, and deputies |
| 6 | Risk management | Identify, evaluate, treat, and monitor information security risk |
| 7 | Performance, risk, and compliance monitoring | Measure effectiveness, risk exposure, and compliance status |
| 8 | Documentation | Control policies, procedures, records, and evidence |
| 9 | Communication | Ensure relevant security messages reach the right audiences |
| 10 | Awareness | Build knowledge, behavior, and security culture |
| 11 | Supplier relationships | Manage third-party and supply-chain security risks |
| 12 | Internal audit | Provide independent assurance |
| 13 | Incident management | Prepare for, respond to, and learn from security events |
| 14 | Continual improvement | Improve suitability, adequacy, and effectiveness over time |

## How to use this model

Use the model for:

- project planning
- maturity assessments
- management briefings
- internal audit universe definition
- roadmap planning
- responsibility mapping
- document architecture
- training paths
