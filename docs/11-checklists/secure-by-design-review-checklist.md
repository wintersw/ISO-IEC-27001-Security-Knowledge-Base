# Secure by Design Review Checklist

- [ ] customer security outcomes defined
- [ ] secure defaults reviewed
- [ ] MFA and logging available appropriately
- [ ] unsafe legacy options identified
- [ ] misuse cases assessed
- [ ] security requirements have acceptance criteria
- [ ] vulnerability disclosure process exists
- [ ] customer evidence available
- [ ] end-of-life support defined

## Evidence

Retain reviewer, date, findings, and improvement actions.
