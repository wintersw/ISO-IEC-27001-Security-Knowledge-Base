# Control Assurance Review Checklist

- [ ] control objective defined
- [ ] control owner confirmed
- [ ] population defined
- [ ] sample selected
- [ ] design effectiveness tested
- [ ] operating effectiveness tested
- [ ] exceptions reviewed
- [ ] evidence quality assessed
- [ ] findings recorded
- [ ] actions assigned

## Evidence

Retain the completed checklist, supporting records, decisions, and assigned actions.

## How to use this checklist

Use this checklist for **Control Assurance Review**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **Control Assurance Review**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **Control Assurance Review**. Evidence retention and independent review of high-risk failures are good practice.
