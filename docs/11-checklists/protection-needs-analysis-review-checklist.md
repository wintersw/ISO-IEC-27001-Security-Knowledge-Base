# Protection Needs Analysis Review Checklist

- [ ] assets and processes identified
- [ ] owners assigned
- [ ] CIA ratings completed
- [ ] privacy relevance marked
- [ ] dependencies considered
- [ ] cumulative effects considered
- [ ] risk analysis decision made
- [ ] results approved

## Evidence

Retain the completed checklist, reviewer, date, decisions, and follow-up actions.

## How to use this checklist

Use this checklist for **Protection Needs Analysis Review**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **Protection Needs Analysis Review**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **Protection Needs Analysis Review**. Evidence retention and independent review of high-risk failures are good practice.
