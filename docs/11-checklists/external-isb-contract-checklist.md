# External ISB Contract Checklist

- [ ] tasks defined
- [ ] rights and duties defined
- [ ] confidentiality agreement included
- [ ] direct reporting path defined
- [ ] project involvement defined
- [ ] handover obligations included
- [ ] termination process defined
- [ ] access rights approved

## Evidence

Retain the completed checklist, decisions, approvals, and follow-up actions.

## How to use this checklist

Use this checklist for **External ISB Contract**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **External ISB Contract**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **External ISB Contract**. Evidence retention and independent review of high-risk failures are good practice.
