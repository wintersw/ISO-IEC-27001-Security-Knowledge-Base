# Sustainable information security management system (ISMS) Culture Checklist

- [ ] security values defined
- [ ] leadership behavior visible
- [ ] employees know how to report concerns
- [ ] control usability reviewed
- [ ] training tailored by role
- [ ] survey results reviewed
- [ ] security improvements tracked
- [ ] culture metrics reported to management

## Evidence

Retain the completed checklist, reviewer, date, decisions, and follow-up actions.

## How to use this checklist

Use this checklist for **Sustainable ISMS Culture**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **Sustainable ISMS Culture**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **Sustainable ISMS Culture**. Evidence retention and independent review of high-risk failures are good practice.
