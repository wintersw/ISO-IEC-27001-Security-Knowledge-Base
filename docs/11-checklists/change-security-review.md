# Change Security Review Checklist

- [ ] change owner assigned
- [ ] security impact assessed
- [ ] risk updated if needed
- [ ] rollback plan defined
- [ ] logging/monitoring considered
- [ ] access impact reviewed
- [ ] approval recorded
- [ ] post-change validation completed

## Evidence

Retain the completed checklist or workflow record with date, owner, decisions, and follow-up actions.
