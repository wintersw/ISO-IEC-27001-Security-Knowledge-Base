# Leadership Responsibility Checklist

- [ ] overall responsibility formally accepted
- [ ] security process initiated
- [ ] security objectives approved
- [ ] security strategy approved
- [ ] roles assigned
- [ ] resources allocated
- [ ] information security management system (ISMS) status reviewed regularly
- [ ] risks and consequences understood
- [ ] decisions documented

## Evidence

Retain the completed checklist, decisions, approvals, and follow-up actions.

## How to use this checklist

Use this checklist for **Leadership Responsibility**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **Leadership Responsibility**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **Leadership Responsibility**. Evidence retention and independent review of high-risk failures are good practice.
