# Evidence Preservation Checklist

- [ ] evidence identified
- [ ] collector assigned
- [ ] time synchronized
- [ ] integrity preserved
- [ ] storage protected
- [ ] chain of custody started
- [ ] access restricted
- [ ] retention defined

## Evidence

Retain the completed checklist with reviewer, date, decision, and follow-up actions.

## How to use this checklist

Use this checklist for **Evidence Preservation**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **Evidence Preservation**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **Evidence Preservation**. Evidence retention and independent review of high-risk failures are good practice.
