# Risk Scenario Quality Checklist

- [ ] scenario names asset and unwanted event
- [ ] business impact described
- [ ] threat and vulnerability distinguished
- [ ] existing controls considered
- [ ] owner assigned
- [ ] treatment specific
- [ ] residual risk evaluated
- [ ] monitoring indicators defined
- [ ] review triggers defined

## Evidence

Retain reviewer, date, findings, and improvement actions.

## How to use this checklist

Use this checklist for **Risk Scenario Quality**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **Risk Scenario Quality**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **Risk Scenario Quality**. Evidence retention and independent review of high-risk failures are good practice.
