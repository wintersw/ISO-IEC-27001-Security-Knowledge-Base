# Employee Security Onboarding Checklist

- [ ] role approved
- [ ] non-disclosure agreement (NDA) or confidentiality obligations completed
- [ ] background screening completed where applicable
- [ ] asset issued and recorded
- [ ] multifactor authentication (MFA) enabled
- [ ] least-privilege access approved
- [ ] security awareness assigned
- [ ] manager confirms access need

## Evidence

Retain the completed checklist or workflow record with date, owner, decisions, and follow-up actions.

## How to use this checklist

Use this checklist for **Employee Security Onboarding**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **Employee Security Onboarding**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **Employee Security Onboarding**. Evidence retention and independent review of high-risk failures are good practice.
