# Supplier Security Review Checklist

- [ ] supplier inventory updated
- [ ] risk rating confirmed
- [ ] security questionnaire reviewed
- [ ] contract clauses verified
- [ ] subprocessors reviewed
- [ ] incidents or changes reviewed
- [ ] residual risk approved

## Evidence

Retain the completed checklist or workflow record with date, owner, decisions, and follow-up actions.

## How to use this checklist

Use this checklist for **Supplier Security Review**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **Supplier Security Review**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **Supplier Security Review**. Evidence retention and independent review of high-risk failures are good practice.
