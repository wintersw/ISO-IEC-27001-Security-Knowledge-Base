# ITSM Evidence Quality Checklist

- [ ] record has owner
- [ ] affected service or asset identified
- [ ] approval visible where required
- [ ] timestamps retained
- [ ] security impact captured
- [ ] evidence attachments retained
- [ ] closure outcome documented
- [ ] linked records preserved

## Evidence

Retain the completed checklist with reviewer, date, decisions, and follow-up actions.
