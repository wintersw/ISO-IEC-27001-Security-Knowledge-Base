# Source-to-Project Map

This map shows where cross-cutting topics are explained. It is a navigation aid, not a claim that one source maps directly to one ISO/IEC control. Readers should begin with the business or security question they need to answer, follow the relevant section, and then use its control links, templates, and evidence guidance.

| Project section | Source themes used |
|---|---|
| 25 Data Security Governance | data lifecycle, data as asset, classification, data-centric security, fraud/confidentiality risk |
| 26 Privacy Engineering | PETs, pseudonymization, anonymization, differential privacy, data provenance, privacy in machine learning (ML) |
| 27 Zero Trust | Zero Trust pillars, secure access service edge (SASE), virtual private network (VPN) contrast, multifactor authentication (MFA), microsegmentation, device posture, monitoring |
| 28 Incident and Breach | breach lifecycle, evidence, notification, documentation, lessons learned, cost and response metrics |
| 29 Emerging Trends | artificial intelligence (AI), LLMs, Internet of Things (IoT), 5G/6G, post-quantum, cloud, remote work, blockchain, federated learning |
| 30 Source Research | source inventory, mapping, source-use policy, external reference register |

Topics deliberately overlap because an effective information security management system (ISMS) connects governance, risk, technology, people, and assurance. For example, a data breach may require data classification guidance from section 25, privacy analysis from section 26, response practices from section 28, and governance escalation from section 33.
