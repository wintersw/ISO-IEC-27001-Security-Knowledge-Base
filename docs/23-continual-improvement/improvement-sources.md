# Improvement Sources

Continual improvement depends on recognizing input signals from across the ISMS.

## Common sources

| Source | Typical improvement |
|---|---|
| Internal audit | corrective action, control redesign, evidence improvement |
| External audit | nonconformity closure, certification readiness improvement |
| Incidents | playbook update, monitoring improvement, training need |
| Vulnerabilities | remediation workflow improvement, patch SLA update |
| Risk review | treatment reprioritization, new control selection |
| Supplier review | contract update, monitoring requirement, exit plan |
| Management review | resource allocation, objective change, risk decision |
| User survey | awareness improvement, policy clarity, usability change |
| Control testing | control redesign, population correction, evidence improvement |
| Threat intelligence | risk reassessment, detection use case, control tuning |
| Legal or contractual change | requirement update, policy change, supplier review |
| Cloud architecture change | SoA update, logging baseline, access review scope change |

## Input triage

Classify inputs as:

- nonconformity
- corrective action
- improvement opportunity
- risk change
- policy update
- control change
- training need
- management decision
- exception or waiver
- backlog item

## Best practices

- Use one improvement register for all improvement items.
- Preserve the original source.
- Link to affected risks and controls.
- Use severity and business value to prioritize.
- Track effectiveness, not only completion.
