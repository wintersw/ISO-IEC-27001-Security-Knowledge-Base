# Effectiveness Review

Effectiveness review confirms whether an action achieved the intended outcome.

## Why it matters

Closing a ticket does not prove that risk was reduced or that a nonconformity will not recur.

## Effectiveness review methods

| Method | Example |
|---|---|
| Retesting | Re-test failed access review sample |
| Trend analysis | Check whether overdue vulnerabilities decreased |
| Reperformance | Repeat part of the control to confirm it works |
| Audit sampling | Sample new records after process change |
| Incident analysis | Check whether similar incident recurs |
| Survey | Ask affected users whether new process is clear |
| Metrics | Compare key performance indicator (KPI)/key risk indicator (KRI) before and after improvement |
| Observation | Observe process operation |

## Timing

Review too early and the action may not have had time to work. Review too late and problems may persist unnoticed.

Typical review points:

- 30 days for procedural changes
- 60–90 days for recurring controls
- next quarterly cycle for access reviews
- next audit cycle for systemic issues
- after next tabletop for incident-response improvements

## Evidence

- effectiveness review record
- test result
- sample evidence
- metric trend
- survey results
- auditor conclusion
- management review record

## Practical example

A recurring control failure is entered into the improvement process. The owner analyzes the cause, implements a proportionate change, and verifies after an agreed period that the failure has not recurred.
