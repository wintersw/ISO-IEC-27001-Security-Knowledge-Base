# Continual Improvement Metrics

Improvement metrics should show whether the ISMS learns, adapts, and reduces recurring weakness.

## Suggested metrics

| Metric | Type | Purpose |
|---|---|---|
| Improvement items opened | KPI | Shows input volume |
| Improvement items closed on time | KPI | Shows execution discipline |
| High-priority improvements overdue | KRI | Shows unmanaged exposure |
| Repeat audit findings | KRI | Shows ineffective root-cause analysis |
| Corrective actions verified effective | KPI | Shows quality of closure |
| Average age of improvement items | KRI | Shows backlog health |
| Improvements linked to risk | KPI | Shows risk-based improvement |
| Management decisions overdue | KRI | Shows governance bottleneck |
| Improvements from user feedback | KPI | Shows culture and participation |
| Incidents with lessons learned completed | KPI | Shows learning maturity |

## Dashboard interpretation

- Rising improvement items can be good if detection is improving.
- Rising overdue high-priority items is a warning sign.
- Repeat findings usually indicate weak root-cause analysis.
- High closure rate without effectiveness review may create false confidence.
