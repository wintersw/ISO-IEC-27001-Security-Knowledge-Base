# Improvement Prioritization

Not every improvement can be done immediately. Prioritization ensures that limited resources address the most important work first.

## Prioritization criteria

| Criterion | Question |
|---|---|
| Risk reduction | How much risk is reduced? |
| Urgency | Is there an immediate exposure or deadline? |
| Legal/contractual driver | Is this required by law, contract, or certification finding? |
| Business value | Does it improve customer trust or service reliability? |
| Effort | How much time and cost are required? |
| Dependency | Does other work depend on it? |
| Recurrence | Does it prevent repeat findings or incidents? |
| Evidence value | Does it improve auditability and assurance? |

## Simple scoring model

Score each item 1–5.

| Score | Meaning |
|---|---|
| 1 | low |
| 3 | moderate |
| 5 | high |

Priority score:

```text
(risk reduction + urgency + compliance driver + business value + recurrence + evidence value) - effort
```

## Priority categories

| Category | Meaning |
|---|---|
| P1 | urgent or high-risk; management attention required |
| P2 | important; schedule in current planning cycle |
| P3 | useful; schedule when resources allow |
| P4 | monitor or combine with other work |

## Best practices

- Do not hide high-risk items because effort is high.
- Escalate resource conflicts to management.
- Use management review for prioritization decisions.
- Link improvement priority to risk appetite.
- Reassess priorities after incidents or major changes.

## Practical example

A recurring control failure is entered into the improvement process. The owner analyzes the cause, implements a proportionate change, and verifies after an agreed period that the failure has not recurred.

## Evidence to retain

Retain records showing both design decisions and actual operation, such as:

- improvement source and prioritization decision
- root-cause analysis
- implemented action
- effectiveness review and closure approval
