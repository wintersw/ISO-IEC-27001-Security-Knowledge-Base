# External Information Security Officer Contract Checklist

When an organization uses an external information security officer, the contract must clearly define duties, rights, confidentiality, reporting, and transition.

## Contract topics

- scope of services
- duties and deliverables
- reporting obligations
- access rights
- confidentiality / non-disclosure agreement (NDA)
- conflict-of-interest rules
- availability and response times
- participation in projects
- participation in management review
- incident escalation duties
- documentation ownership
- handover at termination
- data protection requirements
- subcontractor restrictions
- liability and insurance
- termination and transition support

## Operational checklist

- [ ] Role and tasks defined
- [ ] Reporting line to management defined
- [ ] Confidentiality agreement included
- [ ] Access rights approved
- [ ] Conflict-of-interest reviewed
- [ ] Deliverables defined
- [ ] Handover obligations defined
- [ ] Exit process defined
- [ ] Backup contact defined
- [ ] Performance review scheduled

## Evidence

- signed contract
- NDA
- role description
- access approval
- deliverable records
- meeting minutes
- handover records

## How to use this checklist

Use this checklist for **External Information Security Officer Contract**. Assign a reviewer, scope and deadline, and evidence for each answer. Failed items require an owned action, exception, or risk decision; a checked box alone is not evidence.

## Practical example

Before approving **External Information Security Officer Contract**, the reviewer checks an authoritative source, assigns a failed item for remediation, and signs off only after resolution or formal acceptance is evidenced.

## ISO requirement, implementation guidance, and best practice

This checklist is guidance, not prescribed ISO wording. Formal requirements depend on the scoped outcome supported by **External Information Security Officer Contract**. Evidence retention and independent review of high-risk failures are good practice.
