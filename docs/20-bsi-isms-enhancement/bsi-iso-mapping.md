# BSI ISMS.1 to ISO 27001 Mapping
This page maps BSI-inspired ISMS management topics to ISO/IEC 27001 areas.

| BSI-inspired topic | ISO/IEC 27001 relationship |
|---|---|
| Leadership overall responsibility | Clause 5 Leadership |
| Security goals and strategy | Clause 5, Clause 6 objectives |
| Information security guideline / policy | Clause 5 policy, A.5.1 |
| Information security officer | Clause 5 roles, A.5.2 |
| External security officer contract | A.5.19–A.5.20 supplier relationships |
| Security organization structure | Clause 5, A.5.2 |
| Security measures | Clause 6 risk treatment, Annex A, SoA |
| Employee integration | Clause 7 awareness, A.6.3 |
| Integration into business processes | Clause 4 context, Clause 8 operation, A.5.8 |
| Security concept | Clause 6 planning, Clause 8 operation |
| Maintaining security | Clause 9 performance evaluation, Clause 10 improvement |
| Management reports | Clause 9 management review |
| Security process documentation | Clause 7 documented information |
| Economic use of resources | Clause 7 resources, Clause 6 planning |
| Target-group-specific policies | A.5.1, A.6.3 |
| Insurance | Risk treatment option: transfer |

## Use in audits

This mapping helps organizations using both ISO 27001 and BSI/IT-Grundschutz language show how management practices support ISO requirements.


## Practical example

A German manufacturing company certified against ISO/IEC 27001 uses this mapping during an internal readiness review. The ISMS manager traces the Informationssicherheitsbeauftragter (ISB) role to relevant Clause 5 responsibilities and selected Annex A control A.5.2, then assesses the external ISB arrangement against the supplier controls selected in its Statement of Applicability. The mapping supports traceability, while role approvals, contracts, operating records, and assurance results provide evidence that the applicable requirements and controls operate.

## Evidence to retain

Retain records bridging BSI and ISO frameworks, such as:

- the approved mapping version with review date
- audit or customer inquiry responses that reference the mapping
- updates triggered by changes to either the BSI module or ISO/IEC 27001
