# Gap Analysis from German ISMS.1 Source

This page summarizes missing or underdeveloped aspects identified from the German ISMS.1 material.

## Key gaps identified

| Topic | Why it matters | Added content |
|---|---|---|
| Security management threat landscape | ISMS failures create organizational risks before technical risks appear | Added ISMS threat landscape page |
| Security concept | The source treats the security concept as a central planning and documentation artifact | Added security concept guide and template |
| Leadership as visible responsibility | Leadership must initiate, steer, control, and model security | Added leadership responsibility checklist |
| ISB / security officer role | The source strongly emphasizes the information security officer role and direct reporting | Added ISB role profile |
| External security officer | Outsourced ISB arrangements require contract, confidentiality, and transition controls | Added external ISB contract checklist |
| Organizational structure and deputy rules | Roles, communication paths, and substitutions must be defined | Added security organization model |
| Employee integration | Employees must be actively integrated, not only trained | Added employee integration playbook |
| Management reporting | Management reports should include risks, effectiveness, efficiency, prioritized actions, and effort estimates | Added management security report template |
| Security process documentation | Decisions and outputs should be documented, current, confidential, and archived | Added security-process documentation guide |
| Economic resource allocation | Security resources should be risk-based and economically justified | Added resource economics page |
| Higher protection needs | Target-group-specific policies and insurance should be considered for elevated protection needs | Added elevated protection needs guidance |
| BSI-to-ISO mapping | Professionals benefit from mapping BSI-style requirements to ISO clauses and Annex A | Added BSI/ISO mapping page |

## Implementation principle

The additions do not replace ISO/IEC 27001. They enrich the practical ISMS operating model with BSI-inspired management discipline.
