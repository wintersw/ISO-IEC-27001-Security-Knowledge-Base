# Privacy Control Assurance

Privacy controls require testing just like security controls.

## Assurance methods

| Control | Assurance method |
|---|---|
| minimization | sample forms, APIs, datasets |
| retention | check deletion jobs and exceptions |
| masking | test user views and database exports |
| pseudonymization | review mapping protection |
| lineage | trace sample records |
| access | review permissions and logs |
| rights handling | sample requests and response evidence |
| breach readiness | tabletop scenario |
| artificial intelligence (AI) data use | dataset approval and lineage review |

## Evidence quality

Good evidence is complete, dated, attributable, linked to the requirement, and retained under document-control rules.

## Typical evidence

- approved policy, standard, procedure, or architecture record
- risk assessment or design review
- owner and role assignment
- implementation plan
- operating records
- monitoring records
- exception or waiver decisions
- test results
- audit records
- management review decisions

## Practical example

A product team applies this guidance before using personal data for a new feature. It documents necessity, evaluates privacy and re-identification risk, selects safeguards, and tests the result before release.
