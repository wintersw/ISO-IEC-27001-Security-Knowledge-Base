# Minimum ISO 27001 Document Set

This is a practical minimum set. The exact set depends on scope, risk, legal obligations, customer expectations, and auditor sampling.

## Governance

- information security management system (ISMS) scope statement
- information security policy
- roles and responsibilities matrix
- document control procedure
- interested-party and requirements register

## Risk

- risk assessment methodology
- risk register
- risk treatment plan
- Statement of Applicability
- risk acceptance records

## Operation

- asset inventory
- access-control procedure
- incident response procedure
- supplier security procedure
- backup and restore procedure
- change management procedure
- vulnerability management procedure
- secure development procedure where applicable

## Evaluation

- key performance indicator (KPI)/key risk indicator (KRI) dashboard
- evidence register
- internal audit program
- internal audit reports
- management review minutes

## Improvement

- corrective action register
- nonconformity reports
- improvement backlog
- lessons learned records


## Practical example

An ISMS coordinator uses this guidance to select the minimum useful document, assigns an owner and approver, and connects the controlled document to the process records that prove actual operation.

## Evidence to retain

Retain records showing both design decisions and actual operation, such as:

- document owner and approval
- version and change history
- distribution or acknowledgement record
- linked operating evidence

Intent documents are insufficient on their own; retain scoped operating records, approvals, exceptions, and verified follow-up.
