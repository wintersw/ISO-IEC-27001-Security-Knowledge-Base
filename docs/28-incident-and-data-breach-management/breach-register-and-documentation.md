# Breach Register and Documentation

A breach register should preserve the facts, decisions, actions, evidence, and lessons learned.

## Minimum fields

- breach ID
- detection date
- reporter
- affected systems
- affected data
- affected parties
- severity
- containment status
- root cause
- evidence preserved
- decisions made
- notifications made
- corrective actions
- closure date
- effectiveness review

## Documentation quality

Records should be accurate, timely, controlled, and protected.

## Typical evidence

- approved policy, standard, procedure, or architecture record
- risk assessment or design review
- owner and role assignment
- implementation plan
- operating records
- monitoring records
- exception or waiver decisions
- test results
- audit records
- management review decisions
