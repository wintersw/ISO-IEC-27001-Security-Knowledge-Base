# Incident and Breach Metrics

## Suggested metrics

| Metric | Purpose |
|---|---|
| mean time to detect | detection effectiveness |
| mean time to triage | response workflow speed |
| mean time to contain | containment capability |
| time to preserve evidence | forensic readiness |
| breach decision time | notification readiness |
| incidents by source | detection coverage |
| repeat incidents | corrective action quality |
| lessons learned closed | improvement discipline |
| tabletop completion | readiness |
| logging gaps identified | evidence maturity |

## Interpretation

Metrics should drive decisions. A low incident count may mean strong controls, weak detection, or underreporting.

## Typical evidence

- approved policy, standard, procedure, or architecture record
- risk assessment or design review
- owner and role assignment
- implementation plan
- operating records
- monitoring records
- exception or waiver decisions
- test results
- audit records
- management review decisions
